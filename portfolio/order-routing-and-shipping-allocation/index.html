Project Overview

Workflow รวมออเดอร์ จัดส่ง และปิดรอบประจำวันอัตโนมัติ

ออกแบบ workflow automation สำหรับงานรวมออเดอร์จากหลายแอดมิน หลายเพจ และหลายแพลตฟอร์ม เพื่อตรวจสอบรายการให้ถูกต้อง, map สินค้าเข้ารหัสในระบบ, แยกเงื่อนไขการจัดส่ง และเตรียมยอดสรุปก่อนปิดรอบในแต่ละวัน

ปัญหา

ทุกเช้าต้องมีหลายคนช่วยกันรวมออเดอร์จากหลายแหล่งเพื่อให้ทันเวลาปิดรอบ และยังต้องตรวจข้อมูลเองแทบทุกจุด

แนวทาง

ใช้ workflow automation รับข้อมูลตลอด 24/7 แล้วทำ mapping, validation, แยก shipment และเตรียม transaction ให้พร้อมก่อนปิดรอบ

ผลลัพธ์

ลด bottleneck เหลือแค่คนเดียวคอย validate ก่อนส่งยอดให้คลัง และไม่ต้องระดมทีมมานั่งรวมออเดอร์ทุกเช้า

Project Story

สถานการณ์ตั้งต้นคือออเดอร์เข้ามาจากแอดมินหลายคน หลายเพจ และหลายแพลตฟอร์มขาย แต่ละแหล่งมีรูปแบบข้อมูลไม่เหมือนกัน ทีมต้องมานั่ง map ชื่อสินค้าหรือรหัสสินค้ากลับเข้าระบบ, ตรวจสอบราคา, promotion, fee ต่าง ๆ, validate ที่อยู่ และคำนวณค่าส่งแบบ manual จากนั้นยังต้องดูต่อว่าแต่ละออเดอร์มีจำนวนรายการเท่าไร, น้ำหนักรวมเท่าไร, ขนาดสินค้าเป็นแบบไหน และพื้นที่จัดส่งอยู่โซนใด เพื่อเลือกวิธีจัดส่งที่ตรงกับเงื่อนไขในระบบว่าจะส่งแบบขนส่งปกติ, รถขนาดใหญ่ หรือเครื่องบิน บางออเดอร์ยังต้องแตกเป็นหลาย shipping number ตามเงื่อนไขที่ตั้งไว้ในระบบ ทำให้ก่อนปิดรอบในแต่ละวันเกิด workload สูงและพึ่งคนที่รู้ flow เดิมมากเกินไป

Process Demonstration

วางภาพจำลองหรือ flow ของงานตรงนี้ได้ เช่น รวมออเดอร์จากหลายแหล่ง, map SKU, validate ราคาและที่อยู่, แยก shipment ตามเงื่อนไข และสรุปยอดก่อนส่งคลังโดยไม่เปิดข้อมูลลูกค้าจริง

Before

หลายคนต้องมานั่งรวมออเดอร์ทุกเช้า ตรวจที่อยู่ ตรวจราคา แยกวิธีส่ง และแตก shipment แบบ manual เพื่อให้ทันเวลาปิดรอบส่งคลัง

After

ระบบ run ตลอด 24/7 เพื่อรวม transaction, map สินค้า, แยก shipment, validate รายการ และเตรียม output พร้อมใช้ เหลือเพียงคนเดียวคอยตรวจรอบสุดท้ายก่อนส่งยอดให้คลัง

How It Works

01

รับออเดอร์จากหลายแอดมิน หลายเพจ และหลายแพลตฟอร์มเข้าจุดกลางตลอด 24/7

02

map ชื่อสินค้าหรือรหัสที่ input มาให้ตรงกับ SKU หรือรหัสในระบบภายใน

03

validate ราคา โปรโมชัน ค่าหัก ที่อยู่ และเงื่อนไขการจัดส่งตามประเภทสินค้า

04

สร้าง transaction, แตก shipping number ตามน้ำหนักหรือขนาด และสรุปยอดให้พร้อมส่งคลังก่อนปิดรอบ

Confidentiality Note

ผลงานชิ้นนี้เป็น mockup ที่อ้างอิงจาก use case งาน order ops จริง โดยมีการปรับชื่อแพลตฟอร์ม โครงสร้าง transaction และเงื่อนไขด้านขนส่งเพื่อปกป้องข้อมูลลูกค้า
