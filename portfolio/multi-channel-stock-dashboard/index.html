Project Overview

Dashboard ติดตามสต๊อกและยอดขายหลายช่องทางแบบวันต่อวัน

กรณีศึกษาจำลองสำหรับงานรวมข้อมูลขายและสต๊อกจากหลายช่องทาง เช่น marketplace, ERP และไฟล์ Excel ขึ้น dashboard เดียว เพื่อให้ทีมขายและผู้บริหารเห็นภาพรวมรายวันได้เร็วขึ้น

ปัญหา

ข้อมูลกระจายหลายแหล่ง ทำให้ทีมไม่เห็นสต๊อกคงเหลือ ยอดขายรายวัน หรือสินค้าที่ต้องเติมได้ทันเวลา

แนวทาง

รวมข้อมูลเข้าสู่ data model เดียว แล้วทำ dashboard สรุปยอดขาย, stock aging, stock alert และ top movers

ผลลัพธ์

ผู้บริหารเห็นตัวเลขเร็วขึ้น ทีมวางแผนเติมสินค้าได้ดีขึ้น และลดเวลารวมข้อมูลก่อนประชุม

Process Demonstration

วาง GIF หรือ mockup screen ได้ตรงนี้ เช่น หน้า overview, stock alert, top-selling SKU, และ filter ตาม channel หรือ warehouse

Before

แต่ละทีมดึงข้อมูลคนละชุด, ต้องรวมไฟล์มือ, และใช้เวลานานกว่าจะรู้ว่าสินค้าตัวไหนกำลังขาดหรือยอดตก

After

เห็นภาพรวมในหน้าเดียว กรองได้ทันที และมีสัญญาณเตือนสินค้าที่ต้องเติมหรือยอดขายผิดปกติแบบเข้าใจง่าย

Confidentiality Note

ผลงานชิ้นนี้เป็น mockup ที่อ้างอิงจาก use case จริง โดยมีการเปลี่ยนตัวเลข ชื่อสินค้า ช่องทางขาย และโครงหน้ารายงานเพื่อปกป้องข้อมูลลูกค้า
