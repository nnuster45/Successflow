Project Overview

ดึง PO จากอีเมล จับคู่ฐานข้อมูล และเตรียมไฟล์วางแผนการผลิตอัตโนมัติ

กรณีศึกษาจำลองสำหรับงานที่ต้องรับ PO จากลูกค้าทางอีเมลทุกวัน จากนั้นดึงข้อมูลเข้า flow กลาง จับคู่กับฐานข้อมูลและ ERP ภายใน เขียน SQL เพื่อดึงข้อมูลที่เกี่ยวข้อง และ transform ออกมาเป็นไฟล์ Excel สำหรับทีมวางแผนการผลิต

ปัญหา

ทีมต้องเปิดอีเมลเช็ก PO ทุกวัน คีย์ข้อมูลซ้ำหลายระบบ และเสียเวลารวมข้อมูลเพื่อทำ production planning file ก่อนเริ่มวางแผนจริง

แนวทาง

ดึงไฟล์ PO จากอีเมล, parse ข้อมูล, query ฐานข้อมูลด้วย SQL, map กับ ERP/internal master และ transform เป็นโครงข้อมูลสำหรับ Excel planning

ผลลัพธ์

ข้อมูลพร้อมใช้เร็วขึ้น ลดการคีย์ซ้ำ และทำให้ทีมผลิตใช้ planning file ที่สม่ำเสมอจากข้อมูลชุดเดียวกัน

Process Demonstration

วางภาพ mockup หรือ GIF ของ flow ได้ตรงนี้ เช่น email intake, PO parsing, SQL mapping, ERP enrichment และ output ไฟล์ Excel สำหรับ production planning

Before

ต้องเช็กเมลทีละฉบับ ดาวน์โหลดไฟล์ทีละชุด แล้วค่อยดึงข้อมูลจาก DB/ERP มาเทียบเองก่อนทำ planning file ทำให้ช้าและเสี่ยงตกหล่น

After

PO ถูกดึงเข้า flow กลาง จับคู่ข้อมูลอัตโนมัติ และออก planning file ที่พร้อมให้ทีมผลิตใช้งานต่อได้เร็วขึ้นทุกวัน

Confidentiality Note

ผลงานชิ้นนี้เป็น mockup ที่อ้างอิงจาก flow งานจริง โดยมีการเปลี่ยนรูปแบบ PO, master data, query structure และโครง planning file เพื่อหลีกเลี่ยงการเปิดเผยข้อมูลลูกค้า
