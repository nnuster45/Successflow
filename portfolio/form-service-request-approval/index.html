Project Overview

Workflow รับคำขอผ่านฟอร์ม อนุมัติ และกระจายงานต่ออัตโนมัติ

กรณีศึกษาจำลองสำหรับ workflow automation ที่รับคำขอหรือแจ้งปัญหาผ่าน form จากนั้นส่งต่อให้ผู้อนุมัติ ตรวจสอบเงื่อนไข และ assign งานไปยังหัวหน้าทีมหรือผู้รับผิดชอบโดยอัตโนมัติ

ปัญหา

คำขอเข้ามาหลายช่องทาง รายละเอียดไม่ครบ และคนกลางต้องตามอนุมัติและส่งต่อด้วยมือทุกครั้ง

แนวทาง

ใช้ form เป็นจุดรับข้อมูลกลาง แล้วออกแบบ routing ให้ข้อมูลวิ่งผ่านขั้น approve, assign และ notification ตามเงื่อนไข

ผลลัพธ์

ข้อมูลครบขึ้น process ชัดขึ้น และทีมเห็นสถานะคำขอได้ง่ายขึ้นโดยไม่ต้องไล่ตามจากแชต

Process Demonstration

วางภาพ form, approval screen, queue งาน หรือ flow diagram ตรงนี้ได้ เช่น ผู้ใช้กรอกคำขอ > ผู้อนุมัติตรวจสอบ > assign ไปทีมปฏิบัติการ > แจ้งสถานะกลับอัตโนมัติ

Before

ทีมรับคำขอจากหลายแหล่ง รายละเอียดไม่ครบ และต้องคอยคัดลอกข้อมูลจากช่องทางหนึ่งไปอีกช่องทางหนึ่งก่อนเริ่มทำงาน

After

คำขอเริ่มจาก form กลางเดียว มีขั้นอนุมัติชัดเจน และส่งต่อไปยังทีมที่รับผิดชอบพร้อมข้อมูลครบใน flow เดียว

Confidentiality Note

ผลงานชิ้นนี้เป็น mockup ที่อ้างอิงจาก use case งานอนุมัติและกระจายงานจริง โดยมีการเปลี่ยนรายละเอียดฟอร์ม ชื่อทีม และเงื่อนไขการ routing เพื่อปกป้องข้อมูลลูกค้า
